# i18n Conventions

모든 사용자 가시 문자열은 **ARB 파일에서 관리** · `S.of(context).키` 로 접근. 코드 하드코딩 금지. 규약 근거는 [`ADR-016 · i18n 처음부터`](../philosophy/adr-016-i18n-from-start.md) 참조.

---

## 파일 구조

```
lib/core/i18n/
├── app_ko.arb                  # 템플릿 언어 (원문)
├── app_en.arb                  # 영어 번역
├── app_localizations.dart      # 자동 생성 — S 클래스
├── app_localizations_ko.dart   # 자동 생성
└── app_localizations_en.dart   # 자동 생성
```

설정은 `l10n.yaml`:

```yaml
arb-dir: lib/core/i18n
template-arb-file: app_ko.arb        # 한국어를 원문으로
output-localization-file: app_localizations.dart
output-class: S                       # S.of(context) 로 접근
nullable-getter: false                # non-null 만 생성
```

---

## 워크플로우: 새 문자열 추가

1. **`app_ko.arb` 에 키 추가**:
   ```json
   {
     "loginButton": "로그인",
     "@loginButton": {
       "description": "로그인 화면의 CTA 버튼 라벨"
     }
   }
   ```

2. **`app_en.arb` 에도 같은 키 추가**:
   ```json
   {
     "loginButton": "Sign In"
   }
   ```

3. **코드 생성**:
   ```bash
   flutter gen-l10n
   ```

4. **사용**:
   ```dart
   Text(S.of(context).loginButton)
   ```

> ⚠️ **한쪽만 추가하면 빌드 실패**. 양쪽 ARB 모두에 키가 있어야 해요 (의도된 제약).

## ARB 파일 포맷

### 기본 키

```json
{
  "@@locale": "ko",
  "login": "로그인",
  "signUp": "회원가입"
}
```

### 설명 (번역가용)

```json
{
  "@loginButton": {
    "description": "로그인 화면의 CTA 버튼 라벨. 축약형 '로그인' 권장."
  }
}
```

### 파라미터 치환

```json
{
  "welcomeMessage": "안녕하세요, {name}!",
  "@welcomeMessage": {
    "description": "홈 화면 인사말",
    "placeholders": {
      "name": {"type": "String"}
    }
  }
}
```

자동 생성 코드:
```dart
String welcomeMessage(String name) => '안녕하세요, $name!';
```

### 복수형 (ICU format)

```json
{
  "itemCount": "{count, plural, =0{항목 없음} =1{1개 항목} other{{count}개 항목}}",
  "@itemCount": {
    "placeholders": {
      "count": {"type": "int"}
    }
  }
}
```

사용:
```dart
Text(S.of(context).itemCount(5))  // "5개 항목"
Text(S.of(context).itemCount(0))  // "항목 없음"
```

### 날짜 · 숫자 포맷

```json
{
  "lastUpdated": "마지막 업데이트: {date}",
  "@lastUpdated": {
    "placeholders": {
      "date": {
        "type": "DateTime",
        "format": "yMMMd"
      }
    }
  }
}
```

## 키 명명 규칙

네이밍 접두사로 용도 구분. [`naming.md`](./naming.md#i18n-키-명명) 참조.

| 접두사 | 예시 | 용도 |
|--------|------|------|
| (없음) | `login` · `signUp` · `settings` | 일반 UI 텍스트 |
| `validation` | `validationEmailRequired` | 폼 검증 에러 |
| `error` | `errorTimeout` · `errorNetwork` | 네트워크 · 앱 에러 |
| `toast` | `toastCopied` · `toastSaved` | 스낵바 · 토스트 |
| `relativeTime` | `relativeTimeMinutes` · `relativeTimeDays` | 상대 시간 |
| `confirm` · `cancel` · `retry` (없음) | | 확인 · 취소 류 |

### 키 명명 관용

- **짧고 명확**: `loginButton` 대신 `login` (맥락이 화면에서 드러남)
- **축약 피하기**: `btn` · `msg` · `err` 금지. `button` · `message` · `error` 풀어쓰기
- **계층 표현 시 camelCase**: `errorNetwork` (`error_network` 아님)
- **중복 방지**: 동일 표현이 여러 화면에 쓰이면 **하나의 키** 로 — `ok`, `cancel`, `retry` 등

## Screen 에서 사용

```dart
class LoginScreen extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final s = S.of(context);  // ← 지역 alias 관용

    return Scaffold(
      appBar: AppBar(title: Text(s.login)),
      body: Column(children: [
        Text(s.welcomeMessage(userName)),           // 파라미터
        Text(s.itemCount(list.length)),              // 복수형
        Text(s.lastUpdated(DateTime.now())),         // 날짜
        PrimaryButton(label: s.submit, onPressed: ...),
      ]),
    );
  }
}
```

### `final s = S.of(context)` 관용

화면마다 `S.of(context).xxx` 를 여러 번 호출. `final s = S.of(context)` 로 alias 하면 코드 짧아짐. 모든 Screen 에서 이 관용 권장.

## ViewModel 은 i18n 의존 금지

ViewModel 은 `BuildContext` 를 가지지 않아 `S.of(context)` 접근 불가. **code 만 state 에 저장**, Screen 이 번역.

```dart
// ❌ ViewModel 에서 i18n (금지)
class BadViewModel extends StateNotifier<...> {
  Future<void> signIn(BuildContext context) async {  // ← context 받지 말기
    try {
      // ...
    } catch (e) {
      state = state.copyWith(errorMessage: S.of(context).loginFailed);  // ← 금지
    }
  }
}

// ✅ ViewModel 은 code 만
class LoginViewModel extends StateNotifier<...> {
  Future<void> signIn(String email, String password) async {
    try {
      // ...
    } catch (e) {
      state = state.copyWith(errorCode: safeErrorCode(e, fallbackCode: 'LOGIN_FAILED'));
    }
  }
}
```

Screen:
```dart
if (state.errorCode != null) {
  Text(_localizedError(context, state.errorCode!))
}

String _localizedError(BuildContext context, String code) {
  final s = S.of(context);
  switch (code) {
    case 'LOGIN_FAILED': return s.loginFailed;
    // ...
  }
}
```

자세한 건 [`error-handling.md`](./error-handling.md#screen-에서-i18n-변환) 참조.

## MaterialApp 설정

```dart
// lib/app.dart 발췌
MaterialApp.router(
  localizationsDelegates: const [
    S.delegate,
    GlobalMaterialLocalizations.delegate,
    GlobalWidgetsLocalizations.delegate,
    GlobalCupertinoLocalizations.delegate,
  ],
  supportedLocales: S.supportedLocales,  // [ko, en]
  // locale: Locale('ko'),              // 강제 언어 설정 (미지정 시 시스템 언어)
  routerConfig: _router.router,
)
```

## 새 언어 추가

1. `app_{locale}.arb` 파일 추가 (예: `app_ja.arb`)
2. 모든 키 번역
3. `flutter gen-l10n`
4. 확인: `S.supportedLocales` 에 새 locale 포함

**코드 수정 0줄**. ARB 파일 하나 추가 + 자동 생성.

## 자주 하는 실수

### ❌ 하드코딩

```dart
// 금지
Text('로그인')
Text('이메일을 입력해주세요')
```

**올바르게**:
```dart
Text(S.of(context).login)
Text(S.of(context).validationEmailRequired)
```

### ❌ 한쪽 ARB 에만 키 추가

```json
// app_ko.arb 에만 추가
{
  "newKey": "새 기능"
}
// app_en.arb 에는 없음 → flutter gen-l10n 실패
```

**올바르게**: 양쪽에 추가 + `flutter gen-l10n` 실행.

### ❌ `gen-l10n` 안 돌리고 사용 시도

```dart
Text(S.of(context).newKey)  // ← 정의 안 된 키 → 컴파일 에러
```

**올바르게**: ARB 수정 직후 항상 `flutter gen-l10n`.

### ❌ ViewModel 에서 `S.of(context)`

```dart
// 금지
Future<void> doSomething(BuildContext context) async {
  state = state.copyWith(message: S.of(context).success);
}
```

**올바르게**: ViewModel 은 code, Screen 에서 번역.

### ❌ 파라미터 치환 없이 String concat

```json
// 금지
{
  "welcome": "안녕하세요, "
}
```

```dart
Text(S.of(context).welcome + userName)  // 영어엔 " " 앞에, 한국어엔 ", " 뒤에 등 순서 문제
```

**올바르게**: `{name}` 파라미터로.
```json
{
  "welcome": "안녕하세요, {name}!",
  "@welcome": {"placeholders": {"name": {"type": "String"}}}
}
```
